\chapter{Metodologia}
A pesquisa é focada na criação de um artefato tecnológico útil para resolver o problema da fragmentação laboratorial.

\section{Identificação do Problema e Requisitos do Artefato}
O foco é superar a necessidade de conexões físicas por USB e computadores individuais. Os requisitos incluem: Coleta de dados analógicos via ESP32; Transmissão sem fio; Visualização centralizada em Dashboard Web.

\section{Desenvolvimento do Artefato (Arquitetura)}
\begin{itemize}
	\item \textbf{Camada de Borda:} Microcontroladores ESP32 coletando dados de tensão do circuito RC.
	\item \textbf{Camada de Servidor:} Raspberry Pi atuando como servidor central para recepção e persistência dos dados.
	\item \textbf{Interface de Monitoramento:} Dashboard Web desenvolvido para visualização em tempo real \cite{silva}.
\end{itemize}

\section{Demonstração Funcional (PoC)}
A demonstração focará na viabilidade técnica da orquestração:
\begin{itemize}
	\item \textbf{Sincronização de Telemetria:} Demonstração da capacidade da arquitetura em receber dados de múltiplos nós (ESP32) simultaneamente.
	\item \textbf{Fidelidade do Fenômeno:} Coleta da curva de descarga do capacitor e sua exibição correta na interface web.
	\item \textbf{Persistência de Dados:} Verificação da integridade dos dados armazenados no banco de dados para consulta posterior.
\end{itemize}
